## 0. 架构总览与核心不变式

v1.3 在 **v1.2 已就绪的纯函数引擎与状态层** 之上，补齐"人生阶段筛选 / scope 分组 / ❓hover 化 / What-if 一级 Tab / 图旁说明 / 执行清单 / 顶栏清晰化 / 自定义法则"八项能力。

五条贯穿全局的不变式（工程师必须守护）：

1. **引擎纯函数语义不变**：`applyRule(profile, rule, baseOverride?)`、`compoundSeries`、`monthlyDisposable`、`investmentPool` 的返回含义一律不变；v1.3 对它们的修改**仅做加法**（新增解析入口 `getRule` 支持自定义源、`Rule` 加可选字段、新增 `deriveScenarioProfile` 纯函数），不改动既有调用路径的返回含义。
2. **单一事实源**：`UserProfile`（`@core/domain/user.ts`）仍是唯一事实源；所有派生指标（`monthlyDisposable` / `investmentMonthlyOf` / `allocateThreeBuckets`）均为纯函数，禁止在组件中重算一份。
3. **CSS 唯一**：全项目样式仅 `ROOT/src/renderer/styles.css` 一个文件；R7/R1/R3/R6 的新增样式一律落在此文件，**禁止新建 CSS 文件、禁止 CSS-in-JS**。
4. **释义单一来源**：所有术语/起源故事只从 `ROOT/src/renderer/features/education/terms.ts` 经 `getTerm(id)` 取用，组件内禁止硬编码释义（沿用 v1.2 §3.3 / §7.1）。
5. **临时态隔离**：What-if 推演结果**只经 `tempProfile` 内存态注入**，绝不调用 `updateProfile`/`saveProfile`，绝不污染已保存档案（复用 v1.2 设计文档 §7.4）。

### T05 · R4 What-if 场景引擎 `scenario.ts` 纯函数（P1）
- **目标**：定义场景模型与纯函数派生，What-if 通胀率作为**单 Tab 内临时杠杆**（与 E 批排除的 M4 全局通胀开关严格区分，见 O4）。
- **新增 `ROOT/src/renderer/features/simulate/scenario.ts`**
  ```ts
  export interface WhatIfScenario {
    salaryRaisePct: number;     // 年薪涨幅 %
    inflationPct: number;       // 通胀率 %（仅本场景）
    lumpExpense: number;        // 大额支出金额 ¥
    lumpMonth: number;          // 大额支出发生月序号
    unemploymentMonths: number; // 失业月数（前 N 月 income=0）
  }
  export function deriveScenarioProfile(base: UserProfile, s: WhatIfScenario): UserProfile
  ```
  - 年薪涨幅：把 `monthlyIncome` 按 `(1+salaryRaisePct/100)^(months/12)` 复合（总期 `months = investHorizonMonths`，O3 默认口径）。

- **store 衔接**：`WhatIfPanel` 内部 `setTempProfile(deriveScenarioProfile(profile, scenario))`；`setTempProfile` 已存在（`useAppStore.ts:135`），**不新增持久化**。

## 4. 数据流（文字）

### 4.1 R1 阶段筛选 + 推荐随阶段变
```
profile.preset ──> getPreset(preset).recommendRules
     │                         │
     ▼                         ▼
RULES(+自定义) ── 逐条 ──> 三档判定 ──> {推荐置顶, 适合居中, 低关联灰显}
                                  │
                                  ▼
                     getRulesByScope('income'|'invest') 拆两组 + 横线
                                  │
                                  ▼
                          RulesPage 渲染（「为你推荐」标签随 preset 实时算）
```
要点：`getPreset(preset)` 随 `preset` 变化 → `recommendRules` 非静态 → 推荐档与标签天然动态；`preset` 由 `PresetPicker` 写 `profile.preset`（仅影响推荐排序，引擎忽略该字段，延续 v1.2）。

### 4.2 R8 自定义法则入库参与组合
```
CustomRuleModal 保存 ──> saveCustomRule(input)
                              │ (写入 localStorage fb_custom_rules)
                              ▼
RulesPage 读取 loadCustomRules() ──> 并入 12 内置 ──> 三档/双 scope 分组 + 「自定义」标签
                              │
                              ▼
用户选自定义法则 ──> setSelectedRule('custom-xxx')
                              │
                              ▼
PlanPage effective=getRule('custom-xxx') ──> buildPlanView/applyRules
（combine.ts 经 getRule 自动解析，scope 决定进收入层/投资层）
```

### 4.3 R4 What-if 参数隔离时序
```
PlanPage 「What-if 推演」Tab
   │  用户调 年薪涨幅/通胀率/大额支出时间/失业月份（仅本 Tab useState）
   ▼
WhatIfPanel ── deriveScenarioProfile(profile, scenario) ──> setTempProfile(derived)
   │                                                        │（内存态，绝不 updateProfile/saveProfile）
   ▼                                                        ▼
「方案」Tab effective = tempProfile ?? profile ──> buildPlanView(effective, rule) 重算
   │
切走 What-if Tab ──> setTempProfile(null) ──> 基准方案回到真实 profile（不被污染）
```
区分：What-if 通胀率 = 单 Tab 内临时杠杆（不持久、不全局）；E 批排除的 M4 通胀开关 = 全局持久化开关，二者语义不同（O4）。

### 4.4 R5/R6 图表与清单数据同源
```
selectedRuleId ──> getRule ──> rule
   ├─> investmentMonthlyOf(rule, profile) ──> R5 说明行月额 + Visualizer 基数对齐
   └─> applyRule(profile, rule) ∩ INVESTMENT_BUCKET_KEYS ──> R6 投资子分配明细
profile ──> allocateThreeBuckets(profile) ──> R6 备用金/灵活/自由（基于 profile 非法则）
```
不变量：`reserve + flexible + free === monthlyDisposable(profile)`（沿用 `threeBuckets.ts`）。

### 5.1 关于"6 阶段"与任务书的澄清（架构师注记）
任务书 §R1 行文列"6 阶段：单身白领/已婚双职工/三口之家/临退休/退休/自定义"，但 PRD §2 R1 实际采用且代码已落地的 6 阶段为 `PresetId` 枚举 `student|newgrad|single|dualincome|family|preretire`（学生/职场新人/单身白领/已婚双职工/三口之家/临退休）。二者差异处理：

故本设计以 PRD §2 R1 的 `PresetId` 六阶段为唯一事实源，`lifeStages` 取值严格限定于该枚举。
